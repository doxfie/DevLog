# Changelog

## [1.3.2] — 2026-04-30

Hotfix для Telegram-бэкапов: восстановлена отправка только при изменениях в БД и защита от повторного scheduled-бэкапа в тот же день.

### Исправления

- **Smart backup:** проверка `BACKUP_TELEGRAM_SKIP_IF_UNCHANGED` теперь сравнивает логическое содержимое SQLite-таблиц, а не байты свежесозданного `.db`-файла.
- **Scheduler:** добавлен persistent-маркер даты последнего scheduled-запуска, чтобы несколько процессов не отправляли повторный бэкап в пределах одного дня.
- **Telegram-бэкапы:** неизменённая БД больше не приводит к созданию и отправке нового `.db.gz`-архива; старый hash marker автоматически сверяется с последним локальным архивом.

[1.3.2]: https://github.com/doxfie/DevLog/releases/tag/v1.3.2

## [1.3.1] — 2026-04-27

Поддержка SOCKS5-прокси для Telegram-бэкапов, чтобы отправка работала через зарубежный маршрут при сетевых ограничениях.

### Улучшения

- **Telegram-бэкапы:** добавлена переменная `TELEGRAM_PROXY_URL` для отправки Bot API-запросов через SOCKS5-прокси с удалённым DNS (`socks5h://...`).
- **Надёжность отправки:** добавлен настраиваемый таймаут `BACKUP_TELEGRAM_REQUEST_TIMEOUT_MS` и отдельная обработка SOCKS5/TLS/HTTP-ошибок.
- **Документация:** обновлены `.env.example` и README с примером настройки прокси для Telegram.

[1.3.1]: https://github.com/doxfie/DevLog/releases/tag/v1.3.1

## [1.3.0] — 2026-03-09

Telegram-бэкапы с отправкой в чат, улучшения формы и обновлённые визуальные акценты интерфейса.

### Улучшения

- **Бэкапы в Telegram:** добавлены ручной запуск и ежедневный планировщик, отправка SQLite-бэкапа в чат, локальное хранение последних архивов.
- **Формат бэкапа:** отправка в Telegram теперь идёт архивом `.db.gz`; подпись оформлена в компактном стиле с размером, датой и версией приложения.
- **Надёжность подписи:** устранены проблемы кодировки кириллицы в caption Telegram.
- **Форма сессии:** после успешного добавления сессии очищаются поля формы и черновик в localStorage.
- **Интерфейс:** слегка осветлены тёмные тона, подправлен контраст карточек и общий визуальный баланс.

[1.3.0]: https://github.com/doxfie/DevLog/releases/tag/v1.3.0

## [1.2.0] — 2026-03-08

Индикатор длительности сессии, статус паузы во вкладке и favicon.

### Улучшения

- **Форма сессии:** добавлен компактный индикатор «Длительность» в заголовке блока «Сессия»; отображается чистое время (за вычетом перерывов и активной паузы), при активной сессии мигает точка.
- **Вкладка браузера во время паузы:** заголовок обновляется в формате «Перерыв: N мин · DevLog», чтобы не забывать выключать перерыв.
- **Favicon:** добавлены отдельные иконки для обычного режима и перерыва, плюс автоматическое переключение и цвета в стиле интерфейса.

## [1.1.1] — 2026-02-23

Выбор периода в шапке, календарь вместо колёс, упрощение дашборда и интерфейса.

### Улучшения

- **Выбор месяца и недели:** переключатель перенесён в шапку; при клике открывается календарь с выбором недели (клик по дню выделяет всю неделю). Закрытие модалки или кнопка «ОК» применяют выбор; кнопка «Сейчас» переключает на текущую неделю.
- **Дашборд:** блок «Итоги месяца» убран — итоги месяца остаются во вкладке «Дневник».
- **Интерфейс:** шире контейнер приложения (max-width 1200px); кнопки «Текущая неделя» / «Текущий месяц» и «Сейчас» — шрифт 13px; разделители (точки) между месяцем и неделей в шапке и в блоке «О приложении» увеличены.
- **О приложении:** после версии отображается дата релиза через точку (например, 1.1.1 • 23.02.2026).

[1.1.1]: https://github.com/doxfie/DevLog/releases/tag/v1.1.1

## [1.1.0] — 2026-03-02

Исправления дашборда и улучшение отображения итогов.

### Исправления

- **Дашборд, недели:** диапазон запроса сессий для графика «по неделям» теперь заканчивается воскресеньем текущей недели, а не понедельником — часы текущей недели отображаются корректно.
- **Дашборд, месяцы:** исправлен расчёт начала диапазона «последние 12 месяцев» (ошибка индекса месяца в `Date`) — первый месяц и март в частности показывают правильную сумму часов.

### Улучшения

- **Итоги недели и итоги месяца:** каждая строка отображается как пункт списка с маркером (в Дневнике и на Дашборде).
- **Страница входа:** отключено выделение текста у логотипа и подзаголовка.

## [1.0.0] — 2026-02-25

Первый стабильный релиз. Дневник учёбы с авторизацией и деплоем на VPS.

### Возможности

- Учёт сессий (дата, время, перерывы, заметки), черновик формы, удаление с откатом
- Дашборд: графики по неделям/месяцам, KPI-карточки, пороги целей
- Итоги и цели по неделям, итоги месяца
- Настройки: пороги на графиках, блок «О приложении» (версия, статистика)
- Авторизация: один пользователь, логин/пароль из `.env`, сессии (express-session + bcrypt)
- Docker: образ Node 20 Bookworm Slim, порт только на 127.0.0.1
- CI/CD: GitHub Actions — fetch + reset --hard, docker compose up --build
- Документация: первая установка на VPS (Ubuntu 22.04/24.04), Nginx, HTTPS, rate-limit на POST /login, UFW, два ключа (Deploy Key и SSH для Actions)

### Технические требования

- Node.js 20+
- Продакшен: Docker, Nginx, домен для SSL
