# DevLog — краткое описание проекта для макета

## Что это

Веб-приложение **«Дневник учёбы»**: пользователь ведёт учёт учебных сессий (дата, время начала/конца, перерывы в минутах, заметка «что делал»). Система считает чистое время занятий и показывает статистику на дашборде.

**Язык интерфейса:** русский.

---

## Стек на текущий момент

- **Бэкенд:** Node.js, Express, SQLite (better-sqlite3). Один файл БД `devlog.db`, таблица `sessions`.
- **Фронтенд:** ванильный HTML + CSS + JS (без фреймворков). Модули ES (type="module").
- **Графики:** Chart.js (подключён с CDN), столбчатые диаграммы.
- **Хранение на клиенте:** localStorage (черновик формы, состояние таймера паузы).

---

## Структура интерфейса

### Шапка (общая для всего приложения)

- Слева: заголовок «Дневник учёбы» + две вкладки навигации — **«Дневник»** и **«Дашборд»**.
- Справа (только когда активна вкладка «Дневник»): блок «Итоги месяца: X ч Y мин» + переключатель месяца (стрелки ← / → и подпись «Январь 2026»).

### Вкладка «Дневник»

- **Форма «Новая сессия»** — одна строка полей: Дата, Начало (время), Конец (время), Перерывы (мин) + кнопка «Пауза» рядом с перерывами; ниже — поле «Что делал» (textarea); кнопка «Добавить сессию».
- **Таблица сессий** — заголовок «Сессии (Месяц Год)», таблица: колонки Дата, Время сессии, Перерывы, Занимался, Что делал, кнопка удаления (×). Под таблицей — текст «Нет сессий за выбранный период», если записей нет.

### Вкладка «Дашборд»

Три блока подряд, в каждом — заголовок и один график (Chart.js, bar):

1. **Часы по дням** — подпись периода «(Месяц Год)», столбчатый график по дням текущего месяца.
2. **Часы по неделям** — подпись «(последние 3 месяца)», столбчатый график по неделям.
3. **Часы по месяцам** — подпись «(последние 12 месяцев)», столбчатый график по месяцам.

## Визуальный стиль (текущий)

- Тёмная тема: тёмно-серый фон (#1e1e1e), карточки/поля чуть светлее (#2d2d2d, #252525).
- Текст светлый (#e0e0e0), приглушённые подписи (#9e9e9e).
- Акцент: зелёный (#4a9c6d) — кнопки, итоги, столбцы графиков.
- Границы и сетки графиков — нейтральные серые.
- Шрифт: системный (Segoe UI / system-ui), без кастомных шрифтов.

---

## Ключевые сценарии (для понимания макета)

- Пользователь вводит/редактирует сессию в форме; при фокусе в полях даты и времени может подставляться текущее время (Омск).
- Кнопка «Пауза» запускает таймер перерыва; «Стоп» добавляет минуты в поле «Перерывы».
- Переключение вкладок «Дневник» / «Дашборд» без перезагрузки страницы; в шапке при «Дашборде» скрываются итоги месяца и переключатель месяца.
- На дашборде — только графики и подписи периодов, без формы и таблицы.

---

## Ограничения для макета

- Одна страница (SPA-логика): два вида контента переключаются вкладками, без отдельных URL.
- Адаптивность желательна, но изначально ориентир — десктоп (до ~1000px по ширине контента).
- Форма сессии и таблица — основные элементы вкладки «Дневник»; дашборд — три графика один под другим.
